FlowPro: A Flow Propagation Method for Single Community Detection

Social Networks and Information Analysis

João Loureiro

April 16, 2018

Communities

In the study of complex networks, a network is said to have community structure if the nodes of the network can be easily grouped into.

In this paper, it is used a propagation algorithm (FlowPro) that finds the community of a node in a complex network .

There are various applications for this algorithm:

  • Finding web communities;
  • Detecting the structure of social networks;
  • Enumerate attack vectors on a network;
  • The community of a football player.

Flowpro

Most of the approaches found in the literature are centralized, heuristic without a global optimality criterion.

At the same time they require the entire graph structure partitioning the entire given graph into overlapping or non-overlapping communities.

On the contrary, FlowPro, proposes a local algorithm that can be implemented as a fully distributed method that solves the single community detection problem without the knowledge of the entire graph structure.

This makes possible the application of FlowPro in extremely large graphs or in cases where the entire graph is unknown like in most social networks.

Problem Formulation


$G \Rightarrow$ graph

$V \Rightarrow$ set of nodes

$W \Rightarrow$ set of edge weights

$$G=(V,W)$$


$$\text{node } i \in V; \text{ node } j \in V$$
$W(i,j) \Rightarrow$ edge weight between $i$ and $j$

$$W(i,j)=1 \lor W(i,j)=0$$


$$s \in V \Rightarrow \text{initial node}$$
$C(s) \Rightarrow$ set of nodes from $s$ community

$$V \supseteq C(s) \supseteq s$$


$$\text{node } x \in V$$
$d(x) \Rightarrow$ shortest distance between $x$ and $s$

$n(x) \Rightarrow$ neighbours of $x$


$$\text{node } x \in V$$
$p(x) \Rightarrow$ probability that $x$ belongs on $C(s)$

$\rho \Rightarrow$ average ratio of local links to node degree


$$p(x) = \text{ ?}$$
$$p(x) = \rho^{-d(x)}$$
$$p(x) \leq \frac{\sum_{y \in n(x)}p(y)}{|n(x)|}$$

Flow Propagation Algorithm

In each iteration of the main process of FlowPro, the initial node propagates a flow that is shared to its neighbors.

Each node is able to store, propagate to its neighbors and return a part of flow to the initial node.

When the algorithm converges, the stored flow of the nodes that belong on the community of the initial node is generally higher than the stored flow of the rest graph nodes resulting in the requested community.

The $p(x)$ is analogous on stored flow of node $x$.

The stored flow can be used as a belief (rating) of a node to belong on the community and to answer on several questions like find the k-nearest neighbors of the community of a node.

  1. Initial node $s$ propragates a flow.
  2. Flow is shared between neighbors according to the edge weights.
  3. Each node that receives a flow stores half ($S(x)$) for itself.
  4. The other half of the flow ($T(x)$) is sent to its neighbors but only if the flow is greater than a threshold.

The nodes that are closer to $s$ will have high stored flow.

In addition, it takes into account the graph structure in the sense that a node $x$ that have a lot of connections with nodes of high stored flow, will receive also high quantity of flow ($S(x)$).

The nodes $x$ and $y$ have the same distance from $s$ but $S(x) > S(y)$.
So, the node $x$ belongs on $C(s)$ with higher probability than node $y$.

Based on low and high values of $S(x)$, edges are added or removed on $s$.

The goal of the removal and the extension of neighbors of $s$ is:

  • Decrease shortest paths between the nodes that belong on the community of s and s in order to be able to increase their stored flow in the next iterations;
  • Gradually keep the most of the flow to the nodes of the community by removing bridges;
  • Keep the number of neighbors of s balanced.
  1. In the case that $S(x)$ is greater than $\sum(S(n(x)))$, we set it to $\sum(S(n(x)))$.


$$p(x) \leq \frac{\sum_{y \in n(x)}p(y)}{|n(x)|}$$

There exist a bridge (edge: $s ∼ d$) and due to point 5 the reduction of $S(d)$ will be high.
Without this step, $S(d)$ will be less but close to $S(a)$, $S(b)$ and $S(c)$.

  1. We sort the vector of $S$'s in descending order and we compute the differences between adjacent elements of the sorted vector named $DS$.
  2. Let $K$ be the position of global minimum of $DS$.
  3. The community of node $s$ is defined by the first $K$ nodes with highest $S(x)$.

The community finding algorithm converges to a solution (e.g. the last 10 iterations we receive the same community) or the quantity $\frac{T(s)}{\sum_{x \in V} S(x)}$ is lower than a threshold meaning that $S(*)$ has been converged.
